01 · DetectionWhere and when hazards strike

A weighted, spatially-clustered detection engine flags flash-flood and heatwave events across the Saudi grid. Validated against known 2025 events and a spatial-climatology check.

Annual hazard hotspots (2025). Flash-flood risk concentrates over the south-western Asir mountains and Red Sea coast; heatwave risk over the low, hot interior — the mountains stay cool. The pattern emerges from the data alone and matches Saudi Arabia's real hazard geography.
Known 2025 extreme events. The engine reproduces the 23 Aug Jizan extreme rain (254.9 mm), the 19 Aug moisture surge, and the July–August interior heatwaves — red rings mark extreme-severity clusters.

02 · Knowledge GraphWhy hazards form

An interactive causal knowledge graph links indicators, hazards, driving mechanisms (Red Sea Trough, moisture transport, orographic lifting) and real events — with data-driven correlations that recover known physics (e.g. sea-surface temperature ↔ convective instability). 4 of 5 mechanisms are now grounded in verbatim-verified quotes from 6 real, peer-reviewed publications (e.g. de Vries et al. 2013, JGR-Atmospheres, on the Active Red Sea Trough) — not just domain assertions.

03 · ForecastOne day ahead, not just today

A gradient-boosted spatiotemporal model predicts tomorrow's hazard risk from today's indicators (train on Jan–Jun 2025, test on unseen Jul–Dec) — genuine forecasting, not same-day detection. Verified with threshold-independent metrics, known-event checks, and negative controls; a spatial neighbour-context feature and a graph-neural-network variant were also tested and are reported honestly below.

One-day-ahead forecast vs actual outcome. Top row: predicted risk from the day before; bottom row: what actually happened. Heatwave forecasts (right two columns) show strong spatial correspondence. Flash-flood forecasting (left two columns) is harder — the 23 Aug Jizan hotspot is correctly located but under-confident, and the 19 Aug moisture-surge event is missed entirely, consistent with the rarity and localised, convective nature of flash floods.

  • Heatwave: strong, well-calibrated one-day-ahead skill (ROC-AUC 0.971, PR-AUC 0.795 with a spatial-context feature) — calm days stay near-zero risk.
  • Flash flood: real but modest skill (ROC-AUC 0.873) given extreme rarity (0.5% of grid-days) and one year of training data; correctly ranks CAPE as the dominant driver.
  • Dust storm: real, out-of-sample-validated skill (ROC-AUC 0.887) for a similarly rare event (~2% of grid-days); the label itself was built from the detection engine's own already-validated rule rather than a separately invented definition.
  • Also tested, disclosed honestly: a spatiotemporal GNN improved flash-flood spatial recall on one event but degraded calibration overall — kept as a documented research direction, not deployed.

04 · AgentAsk it a question

A DeepSeek function-calling agent wires the forecast model, the literature-grounded causal knowledge graph, and live conditions into one assistant. Every probability, mechanism and citation it states comes from an actual tool call — never invented. This is what makes the knowledge graph functional rather than just browsable: the agent queries it programmatically on every question, the same way a production detection system would.

See real worked examples →
  • Grounded, not generative. The system prompt forbids stating any number or citation without a tool result backing it — verified across 134 unit tests and 4 end-to-end test scenarios, including honest refusal on out-of-range dates and unknown cities.
  • Proven with a live A/B test, not just claimed. Running the same questions with the causal knowledge-graph tool removed showed 4/4 answers lost their mechanism/citation grounding — but crucially, 0/4 hallucinated a replacement; the agent honestly said it lacked a grounded explanation. Full transcripts: agent/ABLATION_REPORT.md.
  • Terrain-aware. Forecasts for mountain cities (Abha, Taif) are flagged as lower-confidence — a genuine model limitation (coarse feature grid understating steep local relief) found and disclosed while building this check, not assumed away.
  • Reflexive self-check. Every forecast is cross-validated against an independent, rule-based detection engine. This caught a real, disclosed model limitation: on 23 Aug (Jizan), the physical preconditions for flash flooding were already elevated the day before the storm, but the model gave only 12.5% — a genuine precursor signal it underweighted. See agent/REFLEXIVE_CHECK_REPORT.md.
  • 3 hazards, not 2. Dust storm was added with full depth — new indicators, a trained and out-of-sample-validated forecast model, and knowledge-graph grounding that reused an existing literature citation rather than inventing one. See agent/DUST_STORM_REPORT.md.
  • Self-audited KG utilization. Only 3 of 11 KG edge types were actually queried by any tool (~20% of edges) — found by our own audit, not an outside critic. A 5th tool, region_risk_tool, was added to use the previously-idle at_risk_of/exposed_to edges, raising usage to ~40% and surfacing a genuine pre-existing KG data-quality gap along the way. See agent/REGION_RISK_REPORT.md.
  • CAP 1.2 standards-compliant alerts (6th tool). cap_alert_tool converts a forecast into a real CAP 1.2 (Common Alerting Protocol, the OASIS standard MAZU's own national framework is built on) XML alert, ready for broadcast/siren/SMS infrastructure — severity from the detection engine's own thresholds, certainty from the reflexive check's model/rule-engine agreement, and status hardcoded to Exercise (never Actual) since this is a historical-dataset demo, not a live feed. See agent/CAP_REPORT.md.
  • Static showcase, by design. GitHub Pages can't run the Python backend or safely hold an API key client-side, so the linked page shows real, verified agent transcripts with their exact tool-call traces rather than a live chat.
